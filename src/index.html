Step 2 — Add to Outlook

  1. Outlook on the web: Click Get add-ins (the puzzle-piece icon) → My add-ins → Add a custom add-in → Add from File… → pick the manifest you just downloaded.
  2. Outlook desktop (Windows/Mac): File → Get Add-ins → My add-ins → Add a custom add-in → Add from File…
  3. Confirm the consent prompt → done.

The version shown next to Prizm Energy ERP in My add-ins after install should match … above. If it doesn't, you uploaded an older copy — clear it and re-download.

Step 3 — Get your API key

  1. Sign in to Prizm ERP → sidebar Outlook → Add-in API keys (production).
  2. Click Generate new key → copy the token shown (it's revealed once).
  3. In Outlook, open any email → Prizm ERP in the top ribbon (or the email's "…" menu after Step 4) → Open Prizm ERP → Settings → paste the key → Test connection.

Step 4 — Pin the icon to the email toolbar (strongly recommended)

Why? Custom-sideloaded add-ins like ours start in the Outlook top ribbon only. Curated partner add-ins (Asana, ClickUp, Teams, OneNote) get pinned to the email's toolbar by Microsoft default; sideloaded ones don't. One-time pin = same UX as Asana.
  1. Open any email.
  2. Click the … in the email toolbar (same row as Reply / Reply all / Forward, far right).
  3. Scroll to the bottom of the menu → click Customize actions.
  4. In the panel that opens, scroll to the Add-ins section → tick the box next to Prizm Energy ERP.
  5. Click Save at the bottom.

Reopen any email — the purple P icon now sits next to Asana / ClickUp on the email toolbar, and the Prizm ERP menu is in the "…" dropdown.

What's new

API key now persists across sessions (roamingSettings)

build —
  • The add-in used to ask for your API key every time you reopened Outlook. Root cause: Outlook on the web treats add-in iframe localStorage as third-party storage and clears it under modern tracking-prevention rules. Switched to Office.context.roamingSettings, which Microsoft persists server-side per (user + add-in) — survives across devices, sessions, browser cache clears and re-installs.
  • One-time migration: if you already had a key in localStorage from an older build, the first load after this deploy copies it up to roamingSettings automatically.
  • Settings now shows a small line telling you which storage tier is active ("Stored in your Outlook profile … persists across devices" vs "Stored in this browser only").
  • Save button waits for the Exchange round-trip before showing "Saved" — no more silent failure.

Reverse flow: create task from a draft (compose mode)

build 91dac10
  • You can now click Prizm ERP → Create Task while drafting a reply or forward, not only when reading an email. The add-in detects compose mode and adapts:
  • Task description is pre-filled from your draft body (your typed text + the quoted original) — edit freely.
  • Recipients in To: and Cc: that match active staff by email are auto-selected as assignees as green chips.
  • Compose-irrelevant checkboxes are hidden (no .eml capture for unsent drafts, no "email assignees" — you're already drafting the email).
  • Context banner switches from From: to To:/Cc: in compose mode.
  • Workflow: write the email + Create Task in the add-in → task logged + task ID anchored in your draft → hit Send in Outlook.

Lookup Sender = full domain intel · Add company / Add contact · official icon

build f8ca72b
  • Lookup Sender rewrite: shows domain registration status (customer / partner / vendor with pills), relationship summary (count of invoices / opportunities / projects / tickets / estimates / contracts), the contact-on-file if any, and recent activity rows.
  • New Add Company and Add Contact buttons. Strict rule enforced: "Add contact" is disabled until a customer matching the email's domain is registered. Backend (POST /contacts) also rejects orphan contacts at the API level.

  • Add-company form takes a website → domain becomes the unique identifier; mark-as-customer / mark-as-supplier flags written into tblprzbusiness_partners.
  • Add-in icon updated to the official Prizm Outlook add-in art (regenerated at 16/32/64/80/128 from the source 256×256).

Troubleshooting

  • If Test connection returns a red banner: confirm the API base URL in Settings is https://ms.prizm-energy.com/MS/outlookapi/bridge (or https://dev.prizm-energy.com/outlookapi/bridge for dev).
  • If you see the old UI after deploy: in the task pane top-right, click ↻ (it bypasses Outlook's iframe cache). Hard refresh Outlook itself (Ctrl+F5) as a last resort.
  • If the Prizm ERP icon isn't showing in the email toolbar (next to Reply/Forward): do Step 4 above. Custom sideloaded add-ins always need this pin.
